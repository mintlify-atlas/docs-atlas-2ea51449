## Prerequisites

Before you start, make sure you have the following:

- **Java 17** — the server requires `eclipse-temurin:17-jdk` or any compatible JDK 17 distribution
- **Maven 3.8+** — for building the project locally
- **Docker** — if you prefer the containerized approach (no local JDK required)
- **A GT06-compatible GPS tracker** — any tracker that speaks the GT06 binary protocol over TCP

<Note>
  The server listens on TCP port **9000** for GPS devices and exposes Spring Boot Actuator endpoints on HTTP port **8080**.
</Note>

## Step 1: Get the source and build

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/romp-gps.git
    cd romp-gps
    ```
  </Step>

  <Step title="Build the JAR">
    ```bash
    mvn clean package -DskipTests
    ```

    This produces `target/ServidorTPC-*.jar` (or similar). The `target/` directory is what the Dockerfile copies from.
  </Step>
</Steps>

## Step 2: Configure the backend URL

Before starting the server, tell it where to forward GPS data. Open `src/main/resources/application.properties`:

```properties application.properties
spring.application.name=ServidorTPC
backend.base-url=http://localhost:8081
```

Replace `http://localhost:8081` with the base URL of your backend service. The server will POST decoded GPS data to `${backend.base-url}/gps/data`.

<Warning>
  If the backend URL is unreachable at runtime, location packets are still decoded and logged, but the HTTP POST will fail. Check your backend service is running before connecting GPS devices.
</Warning>

## Step 3: Start the server

Choose one of the two approaches below.

<CodeGroup>

```bash Maven (local)
mvn spring-boot:run
```

```bash Docker
# Build the image (requires the JAR from Step 1)
docker build -t romp-gps .

# Run the container, mapping both ports
docker run -p 9000:9000 -p 8080:8080 \
  -e BACKEND_BASE_URL=http://your-backend:8081 \
  romp-gps
```

</CodeGroup>

<Tip>
  When running with Docker, override `backend.base-url` at runtime using the environment variable `BACKEND_BASE_URL` (Spring Boot maps `BACKEND_BASE_URL` → `backend.base-url` automatically via relaxed binding).
</Tip>

You should see Spring Boot startup logs followed by:

```
Netty TCP server started on port 9000
```

## Step 4: Connect your GPS device

Configure your GT06-compatible tracker to point its server IP and port to the machine running ROMP-GPS, **TCP port 9000**.

### What happens on connection

<Steps>
  <Step title="Login packet (0x01)">
    The device sends a GT06 login frame containing its BCD-encoded IMEI. The server decodes the IMEI and sends a login ACK response. You will see a console log similar to:

    ```
    Login from IMEI: 359587010124444
    ```
  </Step>

  <Step title="Location packet (0x12)">
    The device sends location frames. The server decodes each frame into a `GPSData` object and logs it:

    ```
    Location received: GPSData{
      imei='359587010124444',
      timestamp='2026-03-17T14:32:00',
      latitud=19.432608,
      longitud=-99.133209,
      velocidad=45,
      gpsValido=true,
      acc=true,
      corteMotor=false
    }
    ```
  </Step>

  <Step title="Heartbeat packet (0x13)">
    Periodic heartbeat frames are acknowledged automatically. The server extracts ACC (ignition) and `corteMotor` (engine cut) status from the terminal info bits and updates the current device state.
  </Step>
</Steps>

## Step 5: Verify data is reaching your backend

The server POSTs a JSON body to `${backend.base-url}/gps/data` for every valid location packet. The payload maps directly to the `GPSData` model:

```json
{
  "imei": "359587010124444",
  "timestamp": "2026-03-17T14:32:00",
  "latitud": 19.432608,
  "longitud": -99.133209,
  "velocidad": 45,
  "gpsValido": true,
  "acc": true,
  "corteMotor": false
}
```

To quickly verify end-to-end delivery without a real backend, spin up a request inspection service and point `backend.base-url` at it:

```bash
# Using a local netcat listener to inspect the raw POST
nc -l 8081
```

Or update `application.properties` to point at a public tool like [httpbin](https://httpbin.org):

```properties application.properties
backend.base-url=https://httpbin.org
```

<Note>
  With `https://httpbin.org` as the base URL, data will be POSTed to `https://httpbin.org/gps/data`. Httpbin will echo the request body back in its response, which you will see in the server logs.
</Note>

## Checking server health

The Spring Boot Actuator health endpoint is available on port 8080:

```bash
curl http://localhost:8080/actuator/health
```

Expected response:

```json
{
  "status": "UP"
}
```
