Romp-GPS ships with a minimal Dockerfile that packages the Spring Boot JAR into an `eclipse-temurin:17-jdk` image. The container exposes two ports: **9000** for GPS device connections (Netty TCP) and **8080** for the Spring Boot HTTP server (REST API and Actuator).

## Dockerfile

```dockerfile Dockerfile
FROM eclipse-temurin:17-jdk

WORKDIR /app

COPY target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java","-jar","app.jar"]
```

| Line | Purpose |
|------|---------|
| `FROM eclipse-temurin:17-jdk` | Base image. Eclipse Temurin is the Adoptium OpenJDK distribution for Java 17. |
| `WORKDIR /app` | Sets the working directory inside the container so subsequent commands run there. |
| `COPY target/*.jar app.jar` | Copies the fat JAR produced by Maven into the image as `app.jar`. |
| `EXPOSE 8080` | Documents the Spring Boot HTTP port. Port 9000 (Netty TCP) is opened at runtime by the application and should also be published with `-p`. |
| `ENTRYPOINT ["java","-jar","app.jar"]` | Starts the application with no additional JVM flags. |

<Note>
The Dockerfile only documents port 8080 with `EXPOSE`. The Netty TCP server binds port 9000 internally; you must publish it explicitly with `-p 9000:9000` when running the container.
</Note>

## Building the image

The Dockerfile copies from `target/`, so you must build the JAR before building the image.

<Steps>
  <Step title="Package the application">
    ```bash
    ./mvnw clean package
    ```
    This produces `target/servidor-tpc-0.0.1-SNAPSHOT.jar`.
  </Step>
  <Step title="Build the Docker image">
    ```bash
    docker build -t romp-gps .
    ```
  </Step>
</Steps>

## Running the container

```bash
docker run -d \
  --name romp-gps \
  -p 9000:9000 \
  -p 8080:8080 \
  romp-gps
```

| Flag | Purpose |
|------|---------|
| `-d` | Run in detached (background) mode. |
| `--name romp-gps` | Assign a name for easy reference with `docker logs`, `docker stop`, etc. |
| `-p 9000:9000` | Publish the Netty TCP port so GPS devices can connect. |
| `-p 8080:8080` | Publish the Spring Boot HTTP port for the REST API and Actuator endpoints. |

### Overriding the backend URL

The default backend URL is `http://localhost:8081` (set in `application.properties`). Override it at runtime with the `BACKEND_BASE_URL` environment variable:

```bash
docker run -d \
  --name romp-gps \
  -p 9000:9000 \
  -p 8080:8080 \
  -e BACKEND_BASE_URL=http://your-backend:8081 \
  romp-gps
```

<Tip>
Spring Boot maps environment variables to properties using relaxed binding. `BACKEND_BASE_URL` maps to the `backend.base-url` property defined in `application.properties`.
</Tip>

## Viewing logs

```bash
docker logs -f romp-gps
```

On a successful start you will see:

```
TCP GPS Server escuchando en puerto 9000
```

## Health check

Spring Boot Actuator exposes a health endpoint at `/actuator/health` on port 8080. Use it as a Docker health check:

```bash
docker run -d \
  --name romp-gps \
  -p 9000:9000 \
  -p 8080:8080 \
  --health-cmd="curl -f http://localhost:8080/actuator/health || exit 1" \
  --health-interval=30s \
  --health-timeout=5s \
  --health-retries=3 \
  romp-gps
```

Check the container health status:

```bash
docker inspect --format='{{.State.Health.Status}}' romp-gps
```

## Docker Compose

The following `docker-compose.yml` starts ROMP-GPS alongside a backend service and wires them together on a private network:

```yaml docker-compose.yml
services:
  romp-gps:
    image: romp-gps
    build: .
    ports:
      - "9000:9000"
      - "8080:8080"
    environment:
      BACKEND_BASE_URL: http://backend:8081
    depends_on:
      - backend
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 5s
      retries: 3
    networks:
      - gps-net

  backend:
    image: your-backend-image:latest
    ports:
      - "8081:8081"
    networks:
      - gps-net

networks:
  gps-net:
    driver: bridge
```

<Note>
Replace `your-backend-image:latest` with the actual image name for your backend service. The `BACKEND_BASE_URL` uses the Docker Compose service name `backend` as the hostname, which resolves automatically on the `gps-net` bridge network.
</Note>

Start the stack:

```bash
docker compose up -d
```

Stop and remove containers:

```bash
docker compose down
```
